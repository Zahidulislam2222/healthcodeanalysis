# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Current `main` branch | Yes |
| Older commits | No |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Email:** muhammadzahidulislam2222@gmail.com

**Please include:**
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment:** Within 48 hours
- **Assessment:** Within 5 business days
- **Critical fix:** Within 7 days
- **Non-critical fix:** Within 30 days

## Scope

**In scope:**
- `scripts/healthcode-api-bridge.php` (REST API plugin)
- `scripts/healthcode-design-system/` (design system plugin)
- `scripts/*.py` (automation scripts)
- `docker/docker-compose.yml` (local dev setup)
- CI/CD pipeline (`.github/workflows/`)

**Out of scope:**
- WordPress core
- Third-party plugins (Elementor, Rank Math, LiteSpeed Cache, etc.)
- The live site (no unauthorized testing)

## Recognition

Security researchers who report valid vulnerabilities will be credited in the fix commit, if desired.

## Responsible Disclosure

Please do not publicly disclose vulnerabilities until a fix has been released.
